信号量机制升级：动态内存管理与状态查询,任务删除接口

[变更内容]

1. 内存管理重构 (Dynamic Memory Management)：
   - 升级 API：新增 `sem_create(uint32_t init_count)` 替代原有的静态初始化 `sem_init`；新增 `sem_delete(sem_t *sem)` 支持资源回收。
   - 动态分配：信号量对象改为运行时在堆区 (Heap) 通过 `malloc` 动态申请，实现了对象生命周期的灵活管理。
   - 安全销毁：`sem_delete` 内部实现了“清场逻辑”，在释放内存前强制唤醒所有在等待队列中的任务，防止产生僵尸任务。

2. 新增状态查询功能 (Status Query)：
   - 新增 API：`sem_get_info(sem_t *sem, sem_info_t *info)`。
   - 数据结构：引入 `sem_info_t` 结构体，用于快照返回。
   - 功能描述：支持实时查询信号量的“当前剩余资源数 (counter)”和“等待任务数量 (waiting_tasks)”。
   - 线程安全：查询内部采用关中断 (Critical Section) 保护，确保在读取计数器和链表大小时的数据一致性。

3. 应用层适配：
   - `main.c` 中的信号量句柄由结构体实体 (`sem_t`) 升级为指针 (`sem_t *`)。
   - 增加了内存申请失败的判空检查 (NULL Check)。
   - 更新测试逻辑：KEY1 用于释放信号量，KEY2 用于打印当前信号量状态快照，辅助验证阻塞机制的正确性。

[技术意义]
完成了内核对象从静态到动态管理的架构升级，并提供了运行时状态监控手段，显著提升了系统的灵活性与可调试性。
